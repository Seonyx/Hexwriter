# Chapter Review Export — Specification

**Purpose:** Add the ability to export an individual chapter as a printable, double-spaced ODT file for editorial review. Each paragraph is prefixed by its PID so the editor's annotations can be referenced unambiguously back to the source.

**Target stack:** .NET Framework 4.8 / ASP.NET MVC 5 / EF6 / SQL Server. No third-party library required.

---

## 1. Format decision: ODT

ODT (OpenDocument Text) is chosen over PDF for the following reasons:

- **Aligns with the XML-everywhere principle.** ODT is a ZIP package of XML files (`content.xml`, `styles.xml`, `meta.xml`, `META-INF/manifest.xml`, `mimetype`). The format is constructed entirely from the .NET BCL (`System.IO.Compression`, `System.Xml.Linq`). No third-party library and no licensing concerns.
- **Architecturally consistent with the existing EPUB export**, which already handles ZIP+XML packaging.
- **Editors retain all printing and markup options.** ODT opens cleanly in Word 2016+, LibreOffice Writer, and Google Docs. Editors can either print and mark up on paper, or annotate digitally with tracked changes / comments.
- **PDF conversion is one downstream step** (File → Export → PDF) in any of the above applications. PDF is not forfeit; it just isn't generated by the server.

Server-side PDF rendering is **out of scope**. Running LibreOffice headless is not viable on shared IIS hosting; introducing a .NET PDF library is not justified at this stage.

This feature is web-only and does **not** require the dual-interface (class library + CLI wrapper) pattern used for components shared with Openclaw.

---

## 2. Double-spacing: where it belongs

**Double-spacing is baked into the document as a paragraph-style property, not deferred to the print dialog.**

- ODT line spacing is set via `fo:line-height` on the paragraph style and is honoured both on screen and at print time in every compliant reader.
- Print-dialog line spacing is non-portable; many readers (browser viewers, mobile viewers) offer no such control.
- Industry manuscript convention treats double-spacing as a document property; an editor opening the file expects to see it already applied.

---

## 3. Page layout defaults

| Property | Value |
|---|---|
| Page size | A4 (8.27 × 11.69 in) |
| Margins | 1.5 inches on all sides |
| Body font | Times New Roman 12 pt |
| Line spacing | 200% (double) |
| Paragraph indentation | First-line indent 0.5 in; no extra space between paragraphs |
| PID label | Inline at paragraph start, 9 pt, grey (`#666666`), Courier New, in square brackets, followed by a non-breaking space |
| Chapter title | Centred, bold, 14 pt, with 0.5 in bottom margin |
| Header | (none in first iteration) |
| Footer | (none in first iteration; page numbers are an enhancement) |

Margins are deliberately wider than Shunn (1.0 in) to give the editor room for marginal annotations.

The PID label is rendered inline at the start of each paragraph, e.g.:

> [CH01-P-7K2Q9ZPL] The morning sun pierced through the venetian blinds…

Implementation: a `<text:span>` carrying a character style (`PidLabel`) sits inside a `<text:p>` carrying the paragraph style (`ReviewBody`).

---

## 4. Content selection

Per chapter, the exported document contains:

1. Title block: chapter title (book title is metadata only; no header in v1).
2. The chapter's paragraphs in `seq` order, each prefixed by its full PID.
3. Nothing else. Notes, metadata sidecar content, and BookML inline emphasis (italic / bold) are excluded from the first cut.

Notes are intentionally excluded — review copies are for editor comments on prose, not for exposing internal author annotations.

## 5. Module structure

### 5.1 New project

`Seonyx.Export.Review` — class library targeting .NET Framework 4.8. Naming follows the existing `Seonyx.Export.*` convention.

```
Seonyx.Export.Review/
├── ReviewExportService.cs       // orchestrator (public entry point)
├── Model/
│   ├── ChapterReviewModel.cs    // input DTO
│   ├── ParagraphReviewItem.cs   // PID + seq + text
│   └── ReviewExportOptions.cs   // page size, margins, font (with defaults)
└── Odt/
    ├── OdtPackageBuilder.cs     // ZIP assembly
    ├── OdtContentWriter.cs      // content.xml
    ├── OdtStylesWriter.cs       // styles.xml
    ├── OdtMetaWriter.cs         // meta.xml
    ├── OdtManifestWriter.cs     // META-INF/manifest.xml
    └── OdtConstants.cs          // namespace URIs, MIME types
```

### 5.2 Service signature

```csharp
namespace Seonyx.Export.Review
{
    public class ReviewExportService
    {
        /// <summary>
        /// Produces an ODT document for editorial review of a single chapter.
        /// </summary>
        /// <returns>The raw bytes of the .odt file.</returns>
        public byte[] ExportChapter(ChapterReviewModel chapter, ReviewExportOptions options = null);
    }

    public class ChapterReviewModel
    {
        public Guid BookProjectID { get; set; }
        public Guid ChapterID { get; set; }
        public string BookTitle { get; set; }
        public string ChapterTitle { get; set; }
        public IReadOnlyList<ParagraphReviewItem> Paragraphs { get; set; }
        public DateTime GeneratedUtc { get; set; }
    }

    public class ParagraphReviewItem
    {
        public string Pid { get; set; }   // e.g. "CH01-P-7K2Q9ZPL"
        public int Seq { get; set; }
        public string Text { get; set; }  // plain text content
    }

    public class ReviewExportOptions
    {
        public PageSize PageSize { get; set; } = PageSize.A4;
        public double MarginInches { get; set; } = 1.5;
        public string BodyFontFamily { get; set; } = "Times New Roman";
        public double BodyFontSizePt { get; set; } = 12.0;
    }

    public enum PageSize { A4, UsLetter }
}
```

### 5.3 Repository

Add the following to the existing chapter repository (sibling of the methods that feed the EPUB and audiobook exports):

```csharp
ChapterReviewModel GetChapterForReviewExport(Guid bookProjectID, Guid chapterID);
```

Implementation notes:

- Loads paragraphs ordered by `seq` ascending.
- Returns `null` if the chapter does not exist or does not belong to the named `BookProjects` row.
- Populates `BookTitle`, `ChapterTitle`, and `GeneratedUtc` from the project / chapter / current UTC time respectively.
- `Text` is the canonical paragraph text from the decomposed relational store.

## 6. API contract

```
GET /api/books/{bookProjectID}/chapters/{chapterID}/export/review
```

**Success response**

- Status: `200 OK`
- `Content-Type: application/vnd.oasis.opendocument.text`
- `Content-Disposition: attachment; filename="{book-slug}-{chapter-slug}-review.odt"`
- Body: the ODT bytes

`{book-slug}` and `{chapter-slug}` are produced from the titles via the existing slug helper.

**Errors**

- `404 Not Found` — chapter does not exist or does not belong to the named book project
- `401 / 403` — per the existing project authorisation pattern

## 7. UI integration

On the chapter view, add a button or menu item: **"Export for Editorial Review"**.

- Triggers a download of the ODT file (the API endpoint above).
- No options dialog in v1; defaults baked in.
- Tooltip: *"Downloads a double-spaced ODT file with paragraph identifiers, ready for editor markup."*

Place the control next to the existing EPUB / audiobook export controls.

## 8. ODT package reference

### 8.1 ZIP layout

```
mimetype                       (STORED — uncompressed — must be first entry)
META-INF/manifest.xml
content.xml
styles.xml
meta.xml
```

### 8.2 Correctness rules for the ZIP package

These rules are mandatory; readers reject the file if any are violated.

1. The `mimetype` entry **must be the first entry**, **stored (not deflated)**, with **no extra fields** and **no file comment**. Its contents are the literal string `application/vnd.oasis.opendocument.text` with no trailing newline.
2. The remaining entries may be deflated.
3. `META-INF/manifest.xml` lists every file in the package, including `mimetype` itself and the manifest.
4. All XML files declare the OpenDocument namespaces explicitly and use UTF-8 encoding.

### 8.3 Key namespace URIs

| Prefix | URI |
|---|---|
| `office` | `urn:oasis:names:tc:opendocument:xmlns:office:1.0` |
| `style` | `urn:oasis:names:tc:opendocument:xmlns:style:1.0` |
| `text` | `urn:oasis:names:tc:opendocument:xmlns:text:1.0` |
| `fo` | `urn:oasis:names:tc:opendocument:xmlns:xsl-fo-compatible:1.0` |
| `manifest` | `urn:oasis:names:tc:opendocument:xmlns:manifest:1.0` |

### 8.4 `styles.xml` — required style definitions

```xml
<office:document-styles
    xmlns:office="urn:oasis:names:tc:opendocument:xmlns:office:1.0"
    xmlns:style="urn:oasis:names:tc:opendocument:xmlns:style:1.0"
    xmlns:text="urn:oasis:names:tc:opendocument:xmlns:text:1.0"
    xmlns:fo="urn:oasis:names:tc:opendocument:xmlns:xsl-fo-compatible:1.0"
    office:version="1.3">

  <office:styles>

    <style:style style:name="ReviewBody" style:family="paragraph">
      <style:paragraph-properties
          fo:line-height="200%"
          fo:text-indent="0.5in"
          fo:margin-top="0in"
          fo:margin-bottom="0in"/>
      <style:text-properties
          fo:font-family="Times New Roman"
          fo:font-size="12pt"/>
    </style:style>

    <style:style style:name="PidLabel" style:family="text">
      <style:text-properties
          fo:font-family="Courier New"
          fo:font-size="9pt"
          fo:color="#666666"/>
    </style:style>

    <style:style style:name="ChapterTitle" style:family="paragraph">
      <style:paragraph-properties
          fo:text-align="center"
          fo:margin-bottom="0.5in"/>
      <style:text-properties
          fo:font-family="Times New Roman"
          fo:font-size="14pt"
          fo:font-weight="bold"/>
    </style:style>

  </office:styles>

  <office:automatic-styles>
    <style:page-layout style:name="ReviewPage">
      <style:page-layout-properties
          fo:page-width="8.27in"
          fo:page-height="11.69in"
          fo:margin-top="1.5in"
          fo:margin-bottom="1.5in"
          fo:margin-left="1.5in"
          fo:margin-right="1.5in"/>
    </style:page-layout>
  </office:automatic-styles>

  <office:master-styles>
    <style:master-page
        style:name="Standard"
        style:page-layout-name="ReviewPage"/>
  </office:master-styles>

</office:document-styles>
```

### 8.5 `content.xml` — body shape

```xml
<office:document-content
    xmlns:office="urn:oasis:names:tc:opendocument:xmlns:office:1.0"
    xmlns:text="urn:oasis:names:tc:opendocument:xmlns:text:1.0"
    office:version="1.3">

  <office:body>
    <office:text>

      <text:p text:style-name="ChapterTitle">Chapter Title Here</text:p>

      <text:p text:style-name="ReviewBody">
        <text:span text:style-name="PidLabel">[CH01-P-7K2Q9ZPL]&#160;</text:span>The morning sun pierced through the venetian blinds…
      </text:p>

      <text:p text:style-name="ReviewBody">
        <text:span text:style-name="PidLabel">[CH01-P-3M8X2YPL]&#160;</text:span>Across the table, Mateo set down his coffee cup…
      </text:p>

    </office:text>
  </office:body>

</office:document-content>
```

Notes:

- `&#160;` is a non-breaking space, ensuring the PID label and paragraph text never wrap apart.
- Text content is XML-escaped (`&amp;`, `&lt;`, `&gt;`, `&quot;`, `&apos;`) by the writer.

### 8.6 `META-INF/manifest.xml`

```xml
<manifest:manifest
    xmlns:manifest="urn:oasis:names:tc:opendocument:xmlns:manifest:1.0"
    manifest:version="1.3">
  <manifest:file-entry manifest:full-path="/"
                       manifest:media-type="application/vnd.oasis.opendocument.text"
                       manifest:version="1.3"/>
  <manifest:file-entry manifest:full-path="content.xml"
                       manifest:media-type="text/xml"/>
  <manifest:file-entry manifest:full-path="styles.xml"
                       manifest:media-type="text/xml"/>
  <manifest:file-entry manifest:full-path="meta.xml"
                       manifest:media-type="text/xml"/>
</manifest:manifest>
```

### 8.7 `meta.xml`

Minimal — generator string, creation date, and document title (the chapter title).

## 9. Acceptance criteria

1. Calling the API endpoint with a valid `bookProjectID` and `chapterID` returns a `200 OK` with the correct MIME type and a `Content-Disposition: attachment` header naming the file `*-review.odt`.
2. The returned file opens cleanly, with no error or warning dialog, in both:
   - LibreOffice Writer (primary reference renderer)
   - Microsoft Word 2016 or later
3. When opened, the document displays:
   - The chapter title at the top, centred, bold, 14 pt
   - Body paragraphs in `seq` order, double-spaced
   - Each paragraph prefixed by its full PID in square brackets, in 9 pt Courier grey, followed by a non-breaking space and the paragraph text
   - 1.5-inch margins on all sides
   - First-line indentation of 0.5 in on each body paragraph; no extra space between paragraphs
4. No spurious line breaks are introduced within paragraph text by the encoder.
5. The `mimetype` entry is the first entry in the ZIP, stored uncompressed, with no extra fields.
6. A unit test round-trips a fixture chapter through `ReviewExportService.ExportChapter`, opens the resulting bytes as a `ZipArchive`, and verifies:
   - The `mimetype` entry is present, first, stored.
   - `content.xml` contains every input PID in `seq` order.
   - `styles.xml` contains a `ReviewBody` paragraph style with `fo:line-height="200%"`.
7. An integration test exports CTY chapter 1 and writes the file to disk for manual inspection on the first build; thereafter a snapshot comparison guards the byte-for-byte output.
8. A 404 is returned when the chapter does not belong to the named book project.

---

## 10. Out of scope (v1)

- Server-side PDF rendering.
- Multi-chapter or whole-book review export.
- Inline emphasis (italic / bold) from BookML content — v1 renders plain text per paragraph.
- Embedded comments or pre-populated tracked changes.
- Configurable margins, fonts, spacing, or paper size via the UI — defaults are baked in.
- Page-number footers and book-title headers.

---

## 11. Possible follow-ons

- Multi-chapter and whole-book export (sibling endpoint plus a small UI affordance for range selection).
- Optional inline rendering of italic / bold markers from BookML paragraphs.
- Optional inclusion of editor-side notes (gated behind a checkbox) for cases where the author wants the editor to see specific queries.
- Page numbers and a book-title header.
- Configurable margin / font / spacing presets (e.g. "Shunn standard" vs. "Wide-margin markup").
